NEXT TASKS / OPTIONAL ENHANCEMENTS

Project: AI-Powered Eyewear Order Management System
Status: Core requirements are implemented and running locally.
Purpose: This file lists the remaining optional enhancements to make the system production-complete.


CORE REQUIREMENTS STATUS
========================

1. Lens Inventory Management
   Status: COMPLETE
   - Inventory table with power, lens_type, lens_index, coating, quantity.
   - Order creation checks in-house availability and sets inventory_available flag.
   - Prophet demand forecasting drives restock/overstock recommendations.

2. Order Lifecycle & Dashboard
   Status: COMPLETE
   - Full lifecycle: Order Placed -> Inventory Check -> Lens Manufacturing -> Coating -> Frame Assembly -> Quality Check -> Packaging -> Dispatch -> Delivered.
   - QC failure loop: QC Failed -> Rework -> Quality Check.
   - Dashboard shows KPIs, AI Risk Center, status distribution pie chart, inventory intelligence.
   - Orders page supports status updates, delay reason logging, and filtering by status/lens_type/store_location.

3. TAT Prediction & Breach Alerts
   Status: FUNCTIONAL (mock dispatch)
   - RandomForestRegressor predicts completion hours.
   - RandomForestClassifier predicts breach risk score.
   - Alert records are created for high-risk orders and marked "sent" in the DB.
   - Email/WhatsApp dispatch is currently mocked; real providers need to be wired.
